# 前言

## 为什么写这本书

这几年全栈开发的概念被说烂了，但真正能把前后端串起来、再把 AI 揉进去、最后还能部署上线的人，少之又少。不是大家能力不够，而是市面上的资料太散了——Go 的归 Go，React 的归 React，AI 的归 AI，中间那道"怎么把它们捏成一个能跑的系统"的鸿沟，只能靠自己蹚过去。

我写过很多年的业务代码，也带过不少团队。发现一个规律：有基础的人学新技术并不慢，慢的是**不知道怎么把这些技术拼成一个完整的系统**。你可能已经会用 Gin 写接口，也会用 React 写组件，甚至调过 OpenAI 的 API，但让你从零搭一个"带用户系统、带数据库、带 AI 能力、能部署"的平台，还是会发怵。

这本书就是来解决这个问题的。

我们不讲语法基础，不讲"Hello World"。我们直接从一个真实的需求出发，用一套统一的技术栈，把后端、前端、AI、部署整条链路走通。读完这本书，你手里会有一个**能出活的深度研究与报告生成平台**，而且你知道每一行代码为什么这样写。

## 这本书适合谁

- **有 1-3 年开发经验的后端或前端**：想扩展全栈能力，或者想搞清楚 AI 怎么接入业务系统
- **独立开发者/技术负责人**：需要快速搭建 MVP，技术选型要求务实、现代、可维护
- **对 AI 应用开发好奇的工程师**：不只想要一个聊天机器人，想要多智能体协同、RAG、报告生成这类深度应用

如果你完全没写过 Go 或 React，这本书对你来说会有点吃力。但只要你已经用一种语言写过生产代码，就能跟得上。

## 技术栈选型说明

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 后端 | Go 1.26 + Gin | 编译快、并发原生、部署简单、团队好招人 |
| 数据库 | PostgreSQL + pgvector | 关系型 + 向量一套搞定，不用维护两套数据库 |
| 缓存 | Redis | 通用、稳定、团队熟悉 |
| 前端 | React 19 + Node.js 22 | 现代前端标准，Server Components 成熟 |
| UI 组件 | Shadcn UI + Tailwind CSS | 拥有代码而非依赖库，定制自由度高 |
| AI | OpenAI / Claude / DeepSeek / Kimi 原生 API | 不绑框架，理解原理后能换任何模型 |
| 向量检索 | pgvector | PostgreSQL 扩展，与业务数据零延迟 JOIN |
| 部署 | Docker Compose + Kubernetes | 开发用 Compose，生产用 K8s，平滑过渡 |
| CI/CD | GitHub Actions | 与代码仓库一体，社区生态成熟 |

几个刻意的"不选"：
- **不用 ORM**：手写 SQL 在复杂查询和性能调优时更自由，Go 的 `database/sql` + `pgx` 足够好用
- **不用 LangChain/Dify**：用原生 API 封装，理解底层后再用框架，知其然也知其所以然
- **不用单独的向量数据库**：pgvector 在百万级以下场景完全够用，省掉一个组件

## 案例：深度研究与报告生成平台

全书围绕一个真实案例展开：一个帮助用户完成深度研究并自动生成报告的平台。

核心流程：
1. 用户输入研究主题
2. 系统派出多个 AI 智能体（研究员、写手、审核员）分工协作
3. 研究员检索知识库和外部资料，生成摘要
4. 写手基于大纲撰写完整报告
5. 审核员检查事实准确性和逻辑一致性
6. 用户在编辑器中审阅、修改、导出

这个案例覆盖了：**用户系统、权限管理、任务调度、多智能体协同、RAG 检索、长文档处理、实时通信、富文本编辑、版本管理、导出下载**——几乎是一个完整 SaaS 产品的缩影。

## 怎么读这本书

**由浅入深，但允许跳跃**

全书分为七大部分，建议按顺序读，但如果某个技术你已经很熟悉，可以直接跳到下一章：
- 第一部分（第1-5章）偏环境和方法学，可以快速浏览
- 第二、三部分（第6-20章）是前后端核心技术，建议精读
- 第四部分（第21-26章）是 AI 集成，这是本书的重点之一
- 第五部分（第27-31章）是工程化，开发中后期再细读也不迟
- 第六部分（第32-36章）是案例实战，建议跟着代码一起写
- 第七部分（第37-39章）是长期维护的功课，适合团队负责人

**代码即教程**

书中的代码不是片段，而是可以直接运行的。每一章的代码都对应仓库中的一个分支或目录，你可以边读边跑，改改看效果。

**用人话，不堆术语**

我不会说"利用函数式编程范式构建声明式用户界面"，我会说"React 就是让你描述界面长什么样，而不是一步步操作 DOM"。专业术语出现时，一定会配上你能听懂的解释。

## 关于 AI 辅助开发

特别说明第5章。这本书写作期间，AI 编程助手正在爆发式发展。我没有避讳这一点，而是专门用一章讲清楚：**怎么让 AI 成为你的结对编程伙伴，而不是一个你猜不透的黑盒**。包括原型设计、代码生成、调试、重构、审查等各个环节。读完这一章，你会知道什么时候该用 AI、什么时候必须自己思考。

## 关于多智能体

多智能体（Multi-Agent）不是炫技。当单个 AI 的能力受限于上下文长度和专业性时，让多个 AI 各司其职、协同工作，是唯一能产出高质量复杂内容的方式。本书会手把手教你搭建一个 Master-Agent 调度器，让研究员、写手、审核员在你的系统里"开会讨论"。
